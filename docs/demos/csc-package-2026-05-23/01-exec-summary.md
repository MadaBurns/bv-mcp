# BlackVeil DNS complements CSC DomainSec

**One-line positioning.** BlackVeil DNS is the _technical observability surface_ — CT-log subdomain enumeration, dangling-DNS / dangling-CDN detection, off-portfolio brand discovery, per-apex email-auth posture grading. ## Capability map — where the two products sit

The earlier version of this comparison understated CSC. CSC DomainSec is a broad platform covering brand portfolio, subdomain monitoring, third-party registration tracking, and email security. The honest complementary axis is **vantage point** — CSC monitors what CSC manages; BlackVeil enumerates from public observability surfaces (CT logs, RDAP, WHOIS, DNS) that don't depend on the customer having CSC-managed DNS.

|                               | **CSC DomainSec covers** (verified against CSC public DomainSec and Subdomain Monitoring product pages, 2026-05-23)                                                                                                | **BlackVeil DNS adds**                                                                                                                                                                                                                                                                           |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Brand portfolio**           | Defensive registration inventory + registrar consolidation for the _managed_ portfolio; long-term trends in **third-party domain registrations**                                                                                                       | Active enumeration of brand-coincidence apexes (standard TLD/ccTLD set) **with registrar + email-auth posture per apex**, surfaced via RDAP/WHOIS — works without a customer-supplied seed list                                                                                                  |
| **Subdomain monitoring**      | **CSC's solution monitors CSC-managed DNS records daily** + alerts on zone state changes (active ↔ inactive); customer must have CSC-managed DNS for this to apply ([per CSC product page](CSC public Subdomain Monitoring product page)) | **CT-log subdomain enumeration**: picks up every subdomain a certificate has ever been issued for, **regardless of which DNS provider the customer uses** + **two-layer dangling sweep** (DNS NXDOMAIN + application-layer storage-provider not-found) with takeover-claimability classification |
| **Brand monitoring**          | Brand impersonation + phishing takedowns                                                                                                                                                                                                               | Edit-distance + homoglyph + IDN typosquat enumeration with **mail-presence scoring** — every lookalike rated by whether it can phish today (MX/SPF/DMARC posture)                                                                                                                                |
| **Per-apex security posture** | Email security tracking; DNS risks and vulnerabilities; registry locks                                                                                                                                                                                 | **Per-apex letter-graded scan** (25 categories — DMARC, DKIM, SPF, DNSSEC, MTA-STS, BIMI, DANE, TLS-RPT, HTTP security, CAA, SVCB-HTTPS, zone hygiene + more) returned as a reproducible score, not just an alert                                                                                |
| **Off-portfolio discovery**   | Trend reporting on third-party domain registrations                                                                                                                                                                                                    | RDAP/WHOIS resolution exposed as an **API tool** — reproducible per-apex without a CSC managed-portfolio context (the Exabytes-registered `openai.sg` finding is the canonical example)                                                                                                          |

**The line between the products:** CSC's lens is the **customer's managed surface** (registrar portfolio + CSC-hosted DNS + customer-supplied seeds). BlackVeil's lens is the **publicly-observable surface** (anything a certificate has been issued for, anything in registry RDAP, anything in passive DNS). Customers who already have CSC don't need BlackVeil to do what CSC does — they want BlackVeil for the parts CSC structurally can't see, like an asset registered outside the managed portfolio under a different registrant name, or a CT-log certificate issued for a subdomain that the customer's DNS provider doesn't own.

## The case study in one paragraph per axis

**§ Subdomain enumeration + dangling-DNS sweep.** 436 OpenAI + 129 Anthropic + 102 x.ai = **667 unique subdomains** enumerated from Certificate Transparency. Two-layer sweep (DNS NXDOMAIN + application-layer storage not-found) surfaced **14 dangling-pointer findings** across the three named apexes: **1 CRITICAL** (Azure Front Door takeover candidate at `cdn.openai.com` — independently confirmed by the deployed scanner as a "verified takeover signal" via Azure CDN deprovisioned fingerprint), **11 MEDIUM** operational-drift (CloudFront / Azure Container Apps / AWS NLB random-ID targets), **2 LOW** housekeeping pointers. Methodology then re-run against a **22-provider cohort** (the original 10 — Cohere, Nvidia, Mistral, Stability, Perplexity, HuggingFace, Character.AI, DeepMind, Inflection, Meta AI — plus a fresh 12: Databricks, Snowflake, Notion, Stripe, Replicate, Together, Fireworks, Groq, Glean, ElevenLabs, Runway, Midjourney). **21 of 25 providers fully clean (84%)**; the 4 outliers are the original named cases (OpenAI, x.ai) plus Nvidia and HuggingFace — no new findings surfaced by the 12-provider expansion.

**§ Brand impersonation risks.** 68 lookalikes enumerated across the four brand seeds (OpenAI, Anthropic, x.ai, grok.com) via edit-distance, homoglyph, and IDN patterns. **32 mail-active surfaces** (≥1 MX record) — the phishing-readiness signal that matters. Of the 28 verified live on 2026-05-23: **4 fully unprotected** (no SPF + no DMARC — `opena1.com`, `anthropic.co`, `groj.com`, `grol.com`); **21 with no DMARC enforcement** (SPF present but DMARC absent or `p=none`); 3 with DMARC enforcement. **Net phishing-ready: 25 of 28** (89%).

**§ Off-primary-registrar brand sprawl.** Standard TLD-set enumeration with registrar + email-auth posture per domain. **All three brief-named registrars surfaced** (GoDaddy on `openai.io` / `anthropic.co` / `xai.tech` / `grok.ai` / `grok.io`; Network Solutions on `grok.org`; Namecheap on `xai.org`). Plus a less-obvious hit: `openai.sg` on **Exabytes Network (Singapore)**, registrant `PIGEON TECHNOLOGIES PTE. LTD.` — the off-portfolio asset class that pure portfolio inventory misses. **4 fully-spoofable brand apexes** (mail-active, no SPF, no DMARC) — one of which (`openai.org`) is on MarkMonitor, evidence that primary-registrar consolidation is not the same as email-auth hygiene.

## The three vendors — three distinct postures, three different grades

| Vendor        | **Posture grade** | Primary registrar | Defensive sprawl                                     | Email-auth posture                                                                 |
| ------------- | :---------------: | ----------------- | ---------------------------------------------------- | ---------------------------------------------------------------------------------- |
| **OpenAI**    |  **D** (51/100)   | MarkMonitor       | Mixed — 2 owned third-party + 6+ ccTLDs              | 2 fully spoofable + 7 weak-DMARC + 1 explicit non-mail                             |
| **Anthropic** |  **B** (79/100)   | MarkMonitor       | Tight — 1 owned third-party (`anthropic.co`)         | 2 fully spoofable + 4 weak-DMARC + **6 explicit non-mail (gold-standard null-MX)** |

The 4 fully-spoofable brand apexes all grade **F** (26–28/100): `openai.org`, `openai.co.uk`, `anthropic.ca`, `anthropic.co`. Brand-owned third-party apexes range D–D+: `openai.io` (F · 49), `anthropic.io` (D · 52), `openai.sg` (D+ · 62). All grades from the deployed scanner, full per-category breakdown in Walkthrough §0.
